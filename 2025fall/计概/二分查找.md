## 边界的移动
边界移动是防止死循环和遗漏答案的关键。

>[!NOTE]
>由于只有三个值的情况一定能找到准确中点，这样就可以直接化归成两个值的情况
>**所以在考虑循环临界的时候只需要想象两个值的情况：**
>`key`值夹在`l.key`和`r.key`之间，`l，r`相邻
>1. 只往小缩，即`l = mid, r = mid-1` 最后一定是`l == r` 结束，所以`while(l < r)`相等自动退出
>2. 只往大缩与1同理，只不过查找出来的是第一个大于key的值
>3. 一起缩即`l = mid+1, r = mid-1`最后如果设置成`l == r`退出则循环内找不到精确解，所以要让`while(l <= r)`把`mid`找到了`key`的那一步走完，不要漏最后（其实循环外判断也没啥）
### A. 查找**精确目标值**（使用 `while (l <= r)`）
在这种模式下，如果 `clerk[mid]` **不是**目标值，那么 `mid` 本身不可能成为答案，所以可以将边界移动到 `mid` 的**相邻位置**。

|**条件**|**边界移动**|**含义**|
|---|---|---|
|**`id < clerk[mid].id`**|`r = mid - 1`|目标值在左侧，`mid` 已被排除。|
|**`id > clerk[mid].id`**|`l = mid + 1`|目标值在右侧，`mid` 已被排除。|
|**`id == clerk[mid].id`**|`return mid`|找到目标。|

**您代码中的问题：**
您提供的代码片段是**寻找精确目标值**，但它的边界移动是错误的：
```cpp
// ... 原始代码片段 ...
while(l < r){ // 循环条件是 l < r
    int mid = (l+r) / 2;
    if(id < clerk[mid].id) r = mid;     // 错误: 应该排除 mid
    else if(id > clerk[mid].id) l = mid; // 错误: 应该排除 mid
    else return clerk[mid].name;
}
// ...
```
这段代码使用的是 `while (l < r)` 循环条件，但是边界移动却是 `r = mid` 和 `l = mid`。在 `id > clerk[mid].id` 的情况下，如果 `l = mid`，可能会导致 **$l$ 和 $r$ 始终相差 1，陷入死循环**。例如，如果 $l=1, r=2$，那么 $mid=1$。如果条件满足，`l` 仍然等于 1，形成死循环。

**正确的精确查找实现（推荐使用 `l <= r`）：**
```cpp
string find(int id) {
    int l = 1, r = 10;
    while (l <= r) {
        int mid = l + (r - l) / 2;
        if (id < clerk[mid].id) {
            r = mid - 1; // 目标在左侧，排除 mid
        } else if (id > clerk[mid].id) {
            l = mid + 1; // 目标在右侧，排除 mid
        } else {
            return clerk[mid].name; // 找到
        }
    }
    return "NOFOUND"; // 未找到
}
```
### B. 查找**边界**（通常使用 `while (l < r)`）
这种模式用于查找满足某个条件的**最左侧**或**最右侧**的元素。循环结束时，$l$ 和 $r$ 相遇，`l` 即为答案。
#### 示例：查找**第一个大于或等于 $X$ 的元素**
目标是找到第一个满足 `clerk[mid].id >= id` 的位置 $l$。
```cpp
int l = 1, r = 10;
while (l < r) {
    int mid = l + (r - l) / 2; // 中点偏左
    if (clerk[mid].id >= id) {
        r = mid; // mid 可能是答案，所以不能排除 mid，搜索范围缩小到 [l, mid]
    } else {
        l = mid + 1; // mid 肯定不是答案，排除 mid
    }
}
// 循环结束时 l (或 r) 就是答案
```
#### 示例：查找**最后一个小于或等于 $X$ 的元素**
目标是找到最后一个满足 `clerk[mid].id <= id` 的位置 $l$。
```cpp
int l = 1, r = 10;
while (l < r) {
    int mid = l + (r - l + 1) / 2; // 中点必须偏右，防止 l = mid 导致死循环
    if (clerk[mid].id <= id) {
        l = mid; // mid 可能是答案，所以不能排除 mid，搜索范围缩小到 [mid, r]
    } else {
        r = mid - 1; // mid 肯定不是答案，排除 mid
    }
}
// 循环结束时 l (或 r) 就是答案
```
