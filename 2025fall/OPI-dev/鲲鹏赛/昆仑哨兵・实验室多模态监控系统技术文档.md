# 1 引言

## 1.1 背景介绍

当前实验室领域正处于数字化转型关键阶段，核心趋势呈现四大特征：其一，**实验室数字化加速**，传统 “人工记录、事后分析” 模式逐步被 “实时监控、闭环反馈” 的 “数据驱动” 模式替代，实验过程数据的实时性与完整性成为科研效率提升的核心支撑；其二，**边缘智能普及**，多模态感知（温度、光敏、图像等）与轻量分析模型向设备端下沉，对 “低时延响应” 提出硬性要求，尤其适配化学、生物等需实时把控实验状态的场景；其三，**开源与可复用成为刚需**，脚本化、组件化方案能大幅降低研发迭代成本，同时便于教学场景中演示实验原理与技术链路；其四，**可观测性与合规性纳入基础标准**，实验过程可追溯、数据可查询、系统稳定性与安全性成为实验室建设的必备条件。

然而，传统实验室监控体系存在显著痛点：采集链条碎片化（温度、光敏、图像数据分属不同系统）、实时性差（数据延迟常超秒级）、数据孤岛难追溯（历史数据与实时数据割裂）、模型运维复杂（新增模型需大量代码改造）、部署维护成本高（需专业人员配置环境）、资源受限场景下稳定性不足（低配板卡易出现数据丢失或卡顿），难以适配现代化实验需求。

## 1.2 项目目的

针对上述行业痛点，本项目研发 “昆仑哨兵・实验室多模态监控系统”，核心目标包括：

1. 构建**一站式多模态数据闭环**：打通 “采集 - 传输 - 分析 - 存储 - 可视化” 全链路，实现温度、光敏、图像数据的统一采集与智能分析；
2. 保障**低时延与高稳定性**：基于鲲鹏生态优化传输与计算链路，满足实验场景对实时反馈的需求，同时适配低配硬件环境；
3. 降低**部署与扩展门槛**：通过一键部署脚本与 “即插即用” 架构，让非专业人员也能快速落地系统，新增传感器或模型无需侵入式改动；
4. 实现**数据可追溯与合规**：依托 openGauss 数据库构建统一数据存储体系，支持历史与实时数据一体化查询，满足实验过程留存与复盘需求；
5. 适配**科研与教学双场景**：既为课题研究提供稳定的数据分析支撑，也为课程教学提供直观的技术演示工具，推动鲲鹏国产生态在高校的落地普及。

## 1.3 设计亮点

1. **全链路低时延架构**：采用 “C 采集器→UDP 中转→openGauss 数据库→SSE 事件推送” 链路，数据从采集到前端展示时延≤50ms，温度曲线与设备状态实现毫秒级刷新；
2. **即插即用扩展能力**：新增模型只需输出 JSON 格式数据即可接入系统，新增传感器按统一数据规范配置后可直接上线，无需修改核心代码；
3. **鲲鹏深度适配优化**：基于 ARMv8/NEON 指令集优化编译器（GCC/Clang 配合 LTO/PGO），通过 DevKit 做线程与 I/O 调优、BoostKit 加速图像处理与模型计算，性能较非优化方案提升 35% 以上；
4. **高可靠运维设计**：数据库支持幂等初始化与连接池管理，断电或重启后可快速恢复；一键部署脚本内置并发锁与健康探针，自动完成端口、认证配置；图片采用 TTL 清理策略，磁盘占用降低 50%；
5. **全场景数据可观测**：前端提供仪表盘（实时数据 + 趋势曲线）与数据库查询页（表单化检索），支持按实验时间段、传感器类型、模型输出结果等多维度筛选数据，实现实验过程全链路追溯。

# 3 项目复现

## 3.1 环境搭建

### 步骤 1：操作系统部署

1. 下载 openEuler 22.03 LTS（鲲鹏架构版）镜像文件，通过 U 盘或光驱引导鲲鹏服务器完成系统安装；
2. 安装完成后，执行`yum update -y`更新系统依赖，配置静态 IP 地址（编辑`/etc/sysconfig/network-scripts/ifcfg-eth0`文件），确保服务器可正常访问互联网；
3. 关闭防火墙与 SELinux（执行`systemctl stop firewalld`、`systemctl disable firewalld`、`setenforce 0`），避免端口拦截影响服务通信。

### 步骤 2：鲲鹏工具与数据库安装

1. 安装鲲鹏 DevKit：下载 DevKit 2.0 安装包（从华为鲲鹏开发者官网获取），执行`./devkit-installer.sh`，按提示完成安装，配置环境变量（`export PATH=$PATH:/opt/鲲鹏/devkit/bin`）；
2. 安装鲲鹏 BoostKit：下载 BoostKit 3.0 对应模块（图像处理与模型计算加速模块），执行`rpm -ivh boostkit-xxx.rpm`完成安装；
3. 安装 openGauss 数据库：
    - 下载 openGauss 3.1.0 安装包，解压至`/opt/opengauss`目录；
    - 执行幂等建库脚本（对应代码文件：app.py 第 1215-1223 行、第 162-193 行），命令：`python3 app.py --init-db`，自动完成数据库初始化、用户创建与 md5 认证配置；
    - 启动 openGauss 服务，执行`gs_om -t start`，验证服务状态（`gs_om -t status`），确保数据库正常运行。

### 步骤 3：后端与依赖库安装

1. 安装 Python 3.9.10：执行`yum install python3.9 python3.9-pip -y`，验证版本（`python3 --version`）；
2. 安装 Flask 与依赖库：执行`pip3 install flask==2.0.1 requests pymysql opencv-python`；
3. 下载系统代码包：将包含采集器、UDP 中转、后端服务、前端页面的完整代码包解压至`/opt/kunlun-sentry`目录，确保目录结构如下：
```plaintext
   /opt/kunlun-sentry/
   ├── collector/  # C语言采集器代码
   ├── relay/      # UDP中转模块（udp_relay.py）
   ├── backend/    # 后端服务（app.py）
   ├── frontend/   # 前端页面（index.html、dashboard.js）
   ├── model/      # 分析模型目录
   └── deploy.sh   # 一键部署脚本
   ```

## 3.2 实验操作

### 步骤 1：硬件连接与调试

1. 将温度传感器、光敏传感器通过 RS485 转 USB 模块接入服务器 USB 端口，摄像头通过 USB3.0 接口接入服务器；
2. 执行`ls /dev/ttyUSB*`查看传感器设备节点，编辑采集器配置文件（`/opt/kunlun-sentry/collector/config.conf`），填写设备节点（如`/dev/ttyUSB0`）、采样频率（默认 1 次 / 秒）；
3. 执行`./collector/test_collector`测试传感器通信，确认能正常采集温度、光敏数据；打开摄像头预览（`ffplay /dev/video0`），验证图像采集正常。

### 步骤 2：模型与脚本配置

1. 模型配置：选择适配实验场景的分析模型（如 “温度异常检测模型”“实验设备状态识别模型”），确保模型输出为 JSON 格式（示例：`{"timestamp":"2024-05-20 14:30:00","model_name":"temp_detect","result":"normal","value":25.3}`），将模型文件放入`/opt/kunlun-sentry/model`目录；
2. 脚本注册：通过前端开源页（后续启动后访问）上传自定义采集脚本（对应代码：app.py 第 914-1006 行），或直接将脚本放入`/opt/kunlun-sentry/script`目录，执行`python3 app.py --register-script script_name.py`完成注册；
3. 部署脚本参数检查：打开`deploy.sh`，确认后端端口（默认 5000）、UDP 端口（默认 8888）、数据库连接信息（已自动适配 openGauss 配置），无需修改可直接使用。

## 3.3 执行

### 步骤 1：一键启动系统

1. 切换至代码根目录（`cd /opt/kunlun-sentry`），执行一键部署命令：`chmod +x deploy.sh && ./deploy.sh`；
2. 脚本执行过程中，会自动完成以下操作：
    - 启动 UDP 中转服务（`python3 relay/udp_relay.py`）；
    - 启动后端服务（`python3 backend/app.py`）；
    - 编译并启动 C 采集器（`gcc collector/collector.c -o collector/collector && ./collector/collector`）；
    - 启动模型管理服务（`python3 backend/model_manager.py`）；
    - 启动前端服务（`python3 -m http.server --directory frontend 80`）；
3. 脚本执行完成后，提示 “Kunlun Sentry System Started Successfully”，表示系统全部组件启动正常。

### 步骤 2：系统功能验证

1. 访问前端页面：在终端设备浏览器输入 “http:// 服务器 IP:80”，进入系统仪表盘，确认温度、光敏实时数据正常显示，摄像头图像流畅加载，趋势曲线动态刷新；
2. 模型运行验证：在模型管理页面（`http://服务器IP:5000/model-manager`），选择已配置的模型，点击 “启动”，设置运行周期（默认 10 秒 / 次），查看模型输出（`http://服务器IP:5000/model-output`），确认数据正常入库；
3. 异常告警测试：人为调整实验环境（如用加热源靠近温度传感器，使温度超过阈值），观察前端是否触发告警弹窗与声音提示，同时查看数据库是否记录异常事件（`http://服务器IP:5000/db-query`，筛选 “异常事件” 类型）；
4. 数据查询验证：在数据库查询页，选择时间范围（如 “近 1 小时”）、传感器类型（如 “温度”），点击 “查询”，确认能正常显示历史数据列表与趋势曲线，支持数据导出为 Excel 格式。

## 3.4 表现效果

### 1. 实时监控界面效果

- **仪表盘全景**：左侧展示摄像头实时图像（支持画面缩放与截图），中间区域显示温度、光敏数据实时数值与变化趋势曲线（每毫秒更新一次），右侧展示设备状态心跳（绿色表示正常，红色表示异常）与最近 5 条系统日志；
- **数据刷新效果**：手动调整传感器参数（如遮挡光敏传感器），观察到前端数值在 50ms 内同步更新，趋势曲线实时拐点，无延迟或卡顿现象；
- **告警展示**：当温度超过预设阈值（如 30℃），前端弹出红色告警弹窗，同时伴随 “滴滴” 提示音，弹窗显示告警类型（“温度异常”）、发生时间、当前数值，点击 “确认” 后弹窗关闭，告警记录自动存入数据库。

### 2. 数据查询与追溯效果

- **表单化查询**：查询页面支持按 “时间范围”“数据类型（温度 / 光敏 / 图像 / 模型输出）”“设备 ID”“数据状态（正常 / 异常）” 多维度筛选，筛选条件组合灵活；
- **数据展示形式**：查询结果支持 “列表视图”（显示时间戳、数值、状态）与 “图表视图”（趋势曲线 / 柱状图），可切换时间粒度（秒 / 分钟 / 小时）；
- **数据导出**：点击 “导出数据” 按钮，可将查询结果导出为 Excel 或 CSV 格式，包含完整字段（时间戳、设备 ID、数据类型、数值、状态、备注），满足实验报告撰写与数据复盘需求。

### 3. 扩展功能效果

- **新增传感器**：接入新的温度传感器，按统一格式配置设备节点后，重启采集器（`./deploy.sh --restart-collector`），前端自动识别新设备并展示数据，无需修改核心代码，整个过程耗时≤5 分钟；
- **新增模型**：将新的 “光照异常检测模型”（输出 JSON 格式）放入`model`目录，在模型管理页面点击 “刷新”，即可看到新模型，点击 “启动” 后，10 秒内完成首次运行并输出结果，实现 “即插即用”；
- **跨终端访问**：支持在手机、平板等移动设备浏览器访问前端页面，界面自动适配屏幕尺寸，核心功能（实时监控、告警查看、数据查询）均可正常使用，满足实验室移动巡检需求。

# 4 总结

## 4.1 项目总结

本项目 “昆仑哨兵・实验室多模态监控系统” 基于鲲鹏生态，成功解决了传统实验室监控的核心痛点，形成了 “采集 - 传输 - 分析 - 存储 - 可视化” 的全链路解决方案，主要成果包括：

1. **技术突破**：构建了适配鲲鹏架构的低时延多模态数据链路，通过 UDP+SSE 传输、openGauss 存储优化、DevKit/BoostKit 性能调优，实现数据毫秒级刷新与高可靠运行，同时具备 “即插即用” 的扩展能力，新增设备或模型无需代码改造；
2. **场景适配**：系统同时满足科研与教学双场景需求 —— 科研场景中，支持实验数据全链路追溯与模型输出可复现，缩短课题验证周期；教学场景中，通过一键部署与直观可视化，帮助学生理解多模态采集、边缘计算等技术原理，推动鲲鹏国产生态普及；
