**CUDA** 是 **C**ompute **U**nified **D**evice **A**rchitecture（统一计算设备架构）的缩写，它是由 NVIDIA 公司开发的一个并行计算平台和编程模型。

**几个关键概念：**
- **Host (主机)**：指的是你的 CPU 和系统内存（RAM）。这是程序的控制中心，负责启动 GPU 任务。
- **Device (设备)**：指的是你的 NVIDIA GPU 和它的显存（VRAM）。这是执行并行计算的地方。
- **Kernel (核函数)**：这是在 GPU 上执行的核心代码。你可以把它想象成每个 “厨师” 都要执行的那个 “做菜的步骤”。一个 kernel 会被 GPU 上的成千上万个线程并行执行。
- **Thread (线程)**：GPU 上的最小执行单位。每个线程执行 kernel 代码的一个副本。
- **Thread Block (线程块)**：为了方便管理，线程被分组到线程块中。一个线程块中的线程可以通过共享内存（Shared Memory）快速通信。
- **Grid (网格)**：一个 kernel 启动时，会形成一个由多个线程块组成的网格。

**显存**
显存全称显示内存，也叫帧缓存，是显卡的核心部件之一，相当于显卡专属的 “高速数据仓库”，专门用来存储显卡芯片处理过或即将处理的渲染数据，像屏幕像素的颜色、3D 模型的纹理、光照数据以及渲染指令等都储存在这里。后续显卡芯片会快速读取显存中的数据，经运算后转化为图像信号输出到显示器，我们才能看到屏幕上的画面。

**60** ：同代版本的性能参数，越大越强
**Ti**：尾缀标识，表示增强型


#### conda和docker的区别
| 特性       | Conda                                                                                                                          | Docker                                                                                                                         |
| :------- | :----------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------- |
| **核心目标** | **包管理**和**环境隔离**（在同一 OS 内）                                                                                                     | **应用打包**和**环境虚拟化**（跨 OS）                                                                                                       |
| **隔离级别** | **进程级**。所有环境共享同一个操作系统内核和大部分系统资源。                                                                                               | **系统级**。每个容器都有自己独立的文件系统、网络空间、进程树，与宿主机和其他容器严格隔离。                                                                                |
| **隔离边界** | 主要隔离 Python 库和依赖。                                                                                                              | 隔离整个应用运行时环境，包括系统库、配置、甚至不同的 Linux 发行版。                                                                                          |
| **实现方式** | 通过创建独立的目录（环境），并使用 `conda install` 等命令在其中安装包。                                                                                   | 基于操作系统的内核特性（如 Linux Namespaces 和 Cgroups），创建一个轻量级的、隔离的 “虚拟计算机”。                                                                |
| **跨平台性** | 可以在 Windows, macOS, Linux 上运行，但创建的环境**不能直接打包移植**到其他操作系统。                                                                       | 容器镜像可以在任何安装了 Docker 的平台上（Windows, macOS, Linux, 云服务器等）运行，实现了 “**一次构建，到处运行**”。                                                  |
| **资源占用** | **非常轻量**。因为只隔离了必要的库文件，没有重复的操作系统。                                                                                               | **相对较重**。每个容器都包含了一个精简的操作系统文件系统（如一个最小化的 Ubuntu 或 CentOS）。                                                                       |
| **适用场景** | 1. 数据科学、机器学习项目，需要快速切换不同版本的 Python、CUDA、TensorFlow/PyTorch 等。<br><br>2. 开发者在本地管理多个项目的依赖。<br><br>3. 确保项目在团队成员之间的环境一致性（在相同 OS 上）。 | 1. 应用程序的打包和部署，确保开发、测试、生产环境完全一致。<br><br>2. 微服务架构，每个服务独立部署在一个容器中。<br><br>3. 运行需要特定系统库或旧版本操作系统的 legacy 应用。<br><br>4. CI/CD 流程自动化。 |
| **形象比喻** | **一个书房里的多个书架**。你（宿主机）可以在书房里（同一个 OS）拥有多个书架（Conda 环境），每个书架上放着不同的书（库和依赖），你可以随时切换书架来工作。                                            | **多个独立的书房**。你拥有多个完全独立的书房（Docker 容器），每个书房里都有自己的书桌、椅子和书架（完整的运行环境）。你可以进入任何一个书房工作，它们之间互不干扰。                                        |
