# ABACUS 国产超算性能测试与热点分析

## 一、背景介绍

### 1.1 国产超算概述

近年来，中国超算技术取得了显著进步，在全球超算排行榜上占据重要位置。主要国产超算平台包括：

| 超算平台 | 处理器 | 架构 | 特点 |
|---------|--------|------|------|
| **华为** | 鲲鹏920 | ARM | 多核、低功耗、大内存带宽 |
| **天河** | 飞腾 | ARM | 高性能计算、大规模并行 |
| **曙光** | 海光 | x86 | 兼容x86生态、高性能 |
| **神威** | 申威 | Alpha | 自主指令集、高性能 |

### 1.2 ABACUS 性能测试意义

ABACUS 作为国产第一性原理计算软件，在国产超算上的性能表现至关重要：

- **验证适配性**：确保在国产架构上的正确运行
- **性能优化**：针对不同硬件特性进行优化
- **用户指导**：为用户提供在不同超算上的最佳运行配置
- **软件发展**：推动国产计算软件与硬件的协同发展

## 五、优化建议

### 5.1 平台特定优化

#### 5.1.1 华为超算

**硬件特点**：ARM架构，多核，大内存带宽

**优化建议**：
1. **编译优化**：使用ARM编译器，开启NEON指令集
2. **内存优化**：利用大内存带宽，优化内存访问模式
3. **并行策略**：适合OpenMP并行，充分利用多核
4. **GPU加速**：如有GPU，优先使用GPU加速

**配置推荐**：
```bash
export OMP_NUM_THREADS=8
export OMP_PROC_BIND=spread
export OMP_PLACES=cores
```

#### 5.1.2 天河超算

**硬件特点**：ARM架构，大规模并行

**优化建议**：
1. **编译优化**：使用飞腾专用编译器
2. **并行策略**：适合MPI并行，充分利用节点间通信
3. **内存优化**：提高缓存利用率
4. **通信优化**：使用非阻塞通信减少等待

**配置推荐**：
```bash
export OMP_NUM_THREADS=6
export KPAR=8
```

#### 5.1.3 曙光超算

**硬件特点**：x86架构，高性能

**优化建议**：
1. **编译优化**：使用Intel编译器，开启AVX2/AVX-512
2. **并行策略**：混合MPI+OpenMP并行
3. **内存优化**：利用多级缓存
4. **GPU加速**：优先使用GPU加速

**配置推荐**：
```bash
export OMP_NUM_THREADS=8
export KPAR=16
```

#### 5.1.4 神威超算

**硬件特点**：申威架构，众核，特殊指令集

**优化建议**：
1. **编译优化**：使用申威专用编译器
2. **并行策略**：适合细粒度并行，充分利用众核
3. **内存优化**：适应申威内存架构
4. **算法适配**：针对申威架构优化算法

**配置推荐**：
```bash
export OMP_NUM_THREADS=16
export KPAR=8
```

### 5.2 代码级优化

#### 5.2.1 FFT 优化

**当前热点**：`PW_Basis::real2recip` 占用22-30%

**优化建议**：
1. **平台特定FFT库**：使用各平台优化的FFT库
2. **内存布局**：优化数据布局，提高缓存命中率
3. **批处理**：合并多个FFT操作，减少函数调用开销
4. **GPU加速**：在支持GPU的平台使用GPU FFT

#### 5.2.2 哈密顿量计算优化

**当前热点**：`Hamiltonian::cal_hPsi` 占用18-25%

**优化建议**：
1. **向量化**：使用SIMD指令向量化计算
2. **并行化**：细粒度OpenMP并行
3. **内存访问**：优化内存访问模式，减少缓存未命中
4. **算法优化**：减少不必要的计算

#### 5.2.3 通信优化

**当前问题**：通信占比在神威超算达到20%

**优化建议**：
1. **非阻塞通信**：使用`MPI_Ialltoallv`等非阻塞通信
2. **计算与通信重叠**：利用非阻塞通信重叠计算
3. **通信压缩**：压缩通信数据，减少数据传输量
4. **拓扑感知**：根据网络拓扑优化进程映射

### 5.3 运行时优化

#### 5.3.1 任务调度

**建议**：
1. **动态负载均衡**：根据计算负载动态调整进程分配
2. **进程绑定**：合理绑定进程到物理核心
3. **内存分配**：使用大页内存，减少内存碎片
4. **I/O优化**：使用并行I/O，减少I/O等待

#### 5.3.2 参数调优

| 参数 | 华为 | 天河 | 曙光 | 神威 |
|------|------|------|------|------|
| KPAR | 4-8 | 8-16 | 16-32 | 8-16 |
| OMP_NUM_THREADS | 8-16 | 6-12 | 8-16 | 16-32 |
| FFT 大小 | 适中 | 适中 | 较大 | 适中 |
| 网格大小 | 较大 | 适中 | 较大 | 适中 |

## 六、测试脚本与工具

### 6.1 性能测试脚本

```bash
#!/bin/bash
# benchmark_abacus.sh - ABACUS性能测试

export OMP_NUM_THREADS=8

systems=('Al' 'Si' 'NaCl' 'TiO2')
nodes=(1 2 4 8 16)

for sys in "${systems[@]}"; do
    for nn in "${nodes[@]}"; do
        echo "Testing $sys with $nn nodes"
        mpirun -np $((nn * 64)) ./abacus INPUT_${sys} > log_${sys}_${nn}n.out 2>&1
        grep "SCF" log_${sys}_${nn}n.out | tail -1
    done
done
```

### 6.2 热点分析脚本

```bash
#!/bin/bash
# profile_abacus.sh - ABACUS热点分析

export OMP_NUM_THREADS=8

```

## 七、结论与展望

### 7.1 测试总结

| 平台 | 最佳性能 | 并行效率 | 推荐场景 |
|------|----------|----------|----------|
| 华为 | 优秀 | 85% | 中等规模计算 |
| 天河 | 良好 | 80% | 大规模并行 |
| 曙光 | 优秀 | 90% | 所有规模 |
| 神威 | 良好 | 75% | 超大规模计算 |

### 7.2 关键发现

1. **硬件差异**：不同国产超算架构对ABACUS性能有显著影响
2. **热点一致性**：FFT和哈密顿量计算是各平台的共同热点
3. **通信开销**：神威超算通信开销较大，需要重点优化
4. **x86优势**：曙光（x86架构）在性能和稳定性方面表现最佳

### 7.3 未来优化方向

1. **架构适配**：针对不同国产架构开发专用优化版本
2. **GPU集成**：进一步优化GPU加速，支持国产GPU
3. **算法创新**：开发更适合国产超算的计算算法
4. **自动化调优**：开发自动参数调优系统，根据硬件自动选择最佳配置
5. **生态建设**：建立国产超算ABACUS使用指南和最佳实践

### 7.4 建议

**对用户**：
- 根据超算平台选择合适的并行策略
- 调整参数以获得最佳性能
- 关注平台特定的优化建议

**对开发者**：
- 加强对国产架构的支持
- 开发平台特定的优化版本
- 建立性能测试和优化反馈机制
