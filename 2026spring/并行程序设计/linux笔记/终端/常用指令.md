## grep

>**G**lobal **R**egular **Expression** **P**rint
>`“”`可要可不要

- **搜特定的词：** `grep "error" access.log` (把所有带 error 的行显示出来)
- **忽略大小写：** `grep -i "ERROR" access.log` (不管大写还是小写，通通找出来)
- **反向查找：** `grep -v "success" access.log` (找出所有**不包含** success 的行)
- **搜次数：** `grep -c "Error" access.log`（找出所有Error的次数）
- **显示行号** `-n`
- **递归搜索** `-r` （这个搜整个文件夹，其实和`ls -R | grep`差不多）
- **搜关键词附近：** `ls *.py | xargs grep -C 3 "while"`（找所有py文件while前后3行）
### 1. grep + | (管道pipe)

- `ls /usr/bin | grep -i python`
- `python test.py | grep 1`
>[!NOTE] 
>如果需要考虑文件的内容，可以不需要cat，但是需要`xargs`翻译
>例如：`ls *.py | grep -c "while"`就是不行的
>正确的示例：`ls *.py | xargs grep -c "while"`

### 2. grep + 正则表达式寻找

- `grep "a..e" file`：匹配以 a 开头、e 结尾、中间有两个任意字符的词（比如 apple）。
- `grep "^start" file`：匹配以 "start" 开头的行。
- `grep "end$"`：匹配以 "end" 结尾的行。


### find
> find \[在哪里找] \[匹配条件] \[找到后做什么]

- **按文件名找：** 在当前目录（`.`）找名为 `test.txt` 的文件。 `find . -name "test.txt"`
- **忽略大小写：** 找 `Test.txt` 或 `test.txt`。 `find . -iname "test.txt"`
- **模糊匹配：** 找所有 `.log` 结尾的文件。 `find /var/log -name "*.log"`

| **匹配条件**     | **含义**   | **例子**                                     |
| ------------ | -------- | ------------------------------------------ |
| **`-type`**  | **文件类型** | `find . -type d` (只找目录) / `-type f` (只找文件) |
| **`-size`**  | **文件大小** | `find . -size +100M` (找出大于 100MB 的巨型文件)    |
| **`-mtime`** | **修改时间** | `find . -mtime -1` (找 24 小时内修改过的文件)        |
| **`-perm`**  | **权限位**  | `find . -perm 777` (找权限全开的不安全文件)           |
| **`-user`**  | **所有者**  | `find . -user root` (找属于 root 的文件)         |
##### 找到之后：执行动作 (`-exec`)

- **找到并删除：** 删掉所有临时文件。 `find . -name "*.tmp" -delete`
- **找到并详细列出：** `find . -name "*.sh" -ls`
- **找到并执行命令：** 把所有 `.txt` 文件移动到 `backup` 目录。 `find . -name "*.txt" -exec mv {} backup/ \;`
    
>[!拆解：] 
>**它的标准公式：** `-exec [外部指令] {} \;`
>- **`{}`**：这是一个**占位符**，代表 `find` 每次找到的那个文件名。
>- **`\;`**：告诉 `find`，“我的外部命令到此结束”，注意`\`是给`;`的转义符
>- ==有些指令是有快捷方式的比如说`-ls` `-delete`,但是`-exec`是通用的桥梁==
>- `-exec`会对`find`到的每一个文件分别执行一次，所以如果是统计所有文件的字数，行数等等，应该用管道符`|`而非直接接`-exec`

## sed
删除/替换（比awk更简单，但是修改的时候不能重定向到原文件）
> [!NOTE] 
> `sed [选项] '地址定界/命令' 文件名`
#### 选项

| **选项**   | **全称**          | **作用**                                |
| -------- | --------------- | ------------------------------------- |
| **`-n`** | quiet           | 静默模式。不再默认打印所有行，常配合 `p` 命令只打印特定行。      |
| **`-i`** | in-place        | **直接修改文件内容**。不加此选项，修改只停留在屏幕显示。        |
| **`-r`** | regexp-extended | 启用扩展正则表达式（如 `+`, `?`, `()` 等不需要加反斜杠）。 |
| **`-e`** | expression      | 指定多个编辑指令。                             |

#### 地址定界 (Addressing)

>决定你的命令对哪些行起作用：
- **数字**：`10s/...` (只改第10行)；`1,10s/...` (改1到10行)。
- **正则**：`/pattern/s/...` (只在包含 pattern 的行里执行替换)。
- **末尾**：`$d` (删除最后一行)。

#### 命令
##### ① 替换 (Substitute) —— 最常用
**格式：** `s/旧内容/新内容/标记`
- `s/apple/orange/` : 替换每行**第一个** apple。
- `s/apple/orange/g` : **全局**替换（每一行所有的 apple）。
- `s/apple/orange/2` : 只替换每行第 **2** 个 apple。
##### ② 删除 (Delete)
- `sed '3d' file` : 删除第 3 行。
- `sed '1,5d' file` : 删除 1 到 5 行。
- `sed '/^#/d' file` : 删除所有以 `#` 开头的注释行。
##### ③ 插入与追加 (Insert / Append)
- `sed '1i Hello' file` : 在第 1 行**之前**插入一行 "Hello"。
- `sed '$a Bye' file` : 在最后一行**之后**追加一行 "Bye"。
##### ④ 打印 (Print)
- `sed -n '5,10p' file` : **只查看**第 5 到 10 行的内容。

## cp + 被复制文档路径 + 文档复制到的路径

| **参数**   | **含义**               | **为什么要用它？**                      |
| -------- | -------------------- | -------------------------------- |
| **`-r`** | **Recursive (递归)**   | **复制文件夹必带！** 否则无法复制目录及其内部内容。     |
| **`-i`** | **Interactive (交互)** | 如果目标位置已有同名文件，它会**问你**要不要覆盖。安全第一。 |
| **`-p`** | **Preserve (保留)**    | 连带文件的修改时间、权限等“元数据”一起复制过去。        |
你可以一次性把多个文件塞进一个文件夹：
`cp file1.txt file2.txt file3.txt /target/directory/`

## mv
>move 移动 
- 与cp用法类似，可用来重命名
- **改名：** `mv A B`
- **搬家：** `mv A 文件夹/`
- **安全搬家：** `mv -i A 文件夹/`
##### ① 重命名文件 (Rename)
如果在同一个目录下，你把文件“移动”到一个新名字，它就完成了重命名。
- **命令：** `mv old_name.txt new_name.txt`
- **效果：** `old_name.txt` 消失了，取而代之的是 `new_name.txt`。
##### ② 移动文件 (Move)
把文件搬到另一个文件夹里。
- **命令：** `mv file.txt ~/Documents/`
- **效果：** 文件从当前目录消失，出现在了文档文件夹里。
##### ③ 移动并重命名
你可以一边搬家，一边改名。
- **命令：** `mv file.txt ~/Documents/new_file.txt`

|**参数**|**含义**|**建议场景**|
|---|---|---|
|**`-i`**|**Interactive (交互)**|如果目标文件已存在，会**询问**你是否覆盖。最稳妥。|
|**`-n`**|**No-clobber (不覆盖)**|如果目标文件已存在，直接**放弃**操作，绝不覆盖。|
|**`-f`**|**Force (强制)**|不管目标是否存在，直接覆盖，且不提示任何信息（慎用）。|
|**`-v`**|**Verbose (详细)**|显示执行过程，告诉你“xx 已移动到 xx”，心里有个底。|
