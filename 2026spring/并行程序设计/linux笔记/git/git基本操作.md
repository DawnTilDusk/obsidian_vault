## 一、 ⚙️ 初始与基础配置

在使用 Git 前的全局设置，一台电脑通常只需配置一次。

- **配置用户名：** `git config --global user.name "你的用户名"`
- **配置邮箱：** `git config --global user.email "你的邮箱"`
- **查看所有全局配置：** `git config --global --list`
- **克隆远程仓库到本地：** `git clone <远程仓库地址>`

---

## 二、 🔄 日常开发工作流 (核心高频操作)

涵盖了“工作区 -> 暂存区 -> 本地仓库 -> 远程仓库”的完整链条。

|**目标动作**|**Git 命令**|**简要说明**|
|---|---|---|
|**查看状态**|`git status`|查看文件是未跟踪、已暂存还是已修改状态。|
|**添加暂存区**|`git add .`|将当前目录下所有新增/修改的文件加入暂存区。|
|**撤出暂存区**|`git rm --cached <文件名>`|将文件从暂存区退回工作区，以便重新修改。|
|**提交本地仓库**|`git commit -m "提交说明"`|将暂存区修改生成记录，`-m` 必须加，描述修改内容。|
|**推送到远程**|`git push`|将本地代码推送到关联的远程仓库。|
|**首次指定推送**|`git push -u origin <分支名>`|首次推送并关联本地与远程分支，后续可直接 `git push`。|
|**查看差异**|`git diff <文件名>`|查看工作区与暂存区/本地仓库的具体内容增删改差异。|

## 三、 🌿 分支管理 (Branch)

多人协作的核心，隔离开发任务。

- **查看所有分支：** `git branch` （带 `*` 的是当前分支）
- **切换分支：** `git checkout <分支名>`
- **新建并切换分支：** `git checkout -b <新分支名>`
- **合并分支：** `git merge <分支名>` （将目标分支的修改合并到**当前**分支）
- **删除分支：** `git branch -d <分支名>`

> **💡 命名规范建议：** `master/main` (稳定版)、`develop` (开发版)、`feature/xxx` (功能开发)、`bugfix/xxx` (修复漏洞) 。

---

## 四、 ☁️ 远程仓库进阶操作 (Remote)

处理与 GitHub/Gitee 服务器的交互。

- **查看远程仓库地址：** `git remote -v`
- **添加远程关联：** `git remote add <主机别名> <地址>`
- **删除远程关联：** `git remote rm <主机别名>`
- **安全拉取代码：** `git fetch <远程主机名>` （仅拉取不自动合并，需手动 `git merge`）
- **自动拉取并合并：** `git pull origin <分支名>` （相当于 fetch + merge）

## 五、 ⏪ 版本回退与撤销 (Undo & Reset)

写错代码或者提错代码时的“后悔药”。

|**场景需求**|**Git 命令**|**注意事项/说明**|
|---|---|---|
|**撤销单文件修改**|`git checkout <文件名>`|还原某个文件在工作区的修改（不可逆！）。|
|**保留代码回退**|`git reset <commit号>`|HEAD指针回退，但本地工作区代码保留为修改后状态。|
|**彻底清除回退**|`git reset --hard <commit号>`|**⚠️ 慎用！**彻底回退，未提交的修改将被永久删除。|
|**强制覆盖远程**|`git push -f origin <分支名>`|**⚠️ 严禁在公共分支使用！** 会抹除远程仓库的记录。|
|**找回丢失版本**|`git reflog`|如果 reset 错了，用此命令找回原来的 commit 号。|
|**用新提交撤销**|`git revert <commit号>`|保留历史记录，创建一个新提交来“抵消”错误的修改。|
|**单独摘取某次提交**|`git cherry-pick <commit号>`|将某个特定的正确提交应用到当前分支。|

## 六、 📦 其他实用进阶功能

#### 1. 临时存储 (Stash)

写代码到一半突然要切分支修 Bug 时使用。

- **暂存当前修改：** `git stash save "说明"`
- **查看暂存列表：** `git stash list`
- **恢复并继续开发：** `git stash pop`
- **删除暂存：** `git stash drop`
#### 2. 日志查看 (Log)

- **查看完整日志：** `git log`
- **查看最近n条：** `git log -n` （如 `git log -5`）
- **最简洁的一行日志：** `git log --oneline`
- **查看带分支走向的日志：** `git log --graph`
#### 3. 标签管理 (Tag)

用于标记里程碑，如发布 `v1.0`。

- **创建附注标签：** `git tag -a <tag名> -m "说明"`
- **查看所有标签：** `git tag`
- **推送到远程：** `git push origin <tag名>` （或 `--tags` 推送所有）
