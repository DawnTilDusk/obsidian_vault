# 并行程序设计（MPI）第 10 次作业报告

作业内容：MPI 编程（向量点积 / 定积分 / 素数统计）  
代码目录：`/mpi_test/hw_10`

## 1. 实验环境与说明

- 并行库：MPI（`MPI_COMM_WORLD`）
- 语言：C++（使用 `mpicxx` / `mpirun` 运行）
- 计时工具：`MPI_Wtime()`
- 并行总耗时口径：各进程本地耗时取最大值（`MPI_Reduce + MPI_MAX`），以“最慢进程完成时间”代表本次并行运行完成时间

> 说明：以下三题的实现均采用“先做本地计算，再使用 `MPI_Reduce` 汇总”的基本结构；计时与输出尽量避免大量 I/O 干扰计算。

## 2. 题目 1：并行计算向量点积

对应代码：[dot.cpp](file:///mpi_test/hw_10/dot.cpp)

### 2.1 问题与目标

给定两个长度为 `N` 的向量 `A, B`，计算点积：

`A·B = Σ_{i=0..N-1} A[i] * B[i]`

要求使用 MPI 并行计算，并测试不同进程数下运行时间。

### 2.2 数据生成

- 由 `rank==0` 的进程生成整段向量 `A, B`（长度 `N`）
- 使用 `std::mt19937_64` 与均匀分布 `U(0,1)` 生成随机 double
- 为保证可复现，使用固定种子（示例：`123456 + rank`，在本题中只有 root 生成，因此等价于固定 seed）

### 2.3 数据划分与分发

设：

- `base = N / size`
- `rem = N % size`

分发规则：

- `rank < rem` 的进程分到 `base + 1` 个元素
- 否则分到 `base` 个元素

分发方式：

- 由 root 使用点对点通信 `MPI_Send` 将各进程的子数组发送出去
- 各进程使用 `MPI_Recv` 接收自己那段 `A_recv, B_recv`

### 2.4 本地计算与全局规约

- 每个进程计算本地子向量点积：`local_sum = Σ A_recv[i] * B_recv[i]`
- 使用 `MPI_Reduce(&local_sum, &global_sum, 1, MPI_DOUBLE, MPI_SUM, root=0, MPI_COMM_WORLD)` 汇总得到最终点积
- 最终结果仅在 `rank==0` 输出

### 2.5 计时方法与输出字段

本题输出了三类时间：

- `MPI_TIME`：从 `START=MPI_Wtime()` 开始到 `END=MPI_Wtime()` 结束的总耗时（各进程取 `MAX` 得到总并行耗时）
- `VECTOR_TIME`：root 生成随机向量（以及少量打印）所用时间（从 `START` 到生成结束）
- `DOT_TIME`：用 `MPI_TIME - VECTOR_TIME` 粗略估算“分发 + 本地计算 + 规约”的耗时

> 注：`DOT_TIME` 并不是“纯点积计算时间”，它包含通信与规约；如果需要更严格的“仅计算”时间，可在本地循环前后单独计时，并同样用 `MPI_MAX` 汇总。

### 2.6 参数设置

- 向量长度：`N = 10,000,000`

### 2.7 示例运行结果（节选）

| ----- | ------------ | --------------- | ------------ |
| 1     | 2.1006       | 1.6009          | 0.4997       |

## 3. 题目 2：并行计算定积分（梯形法）

对应代码：[integrate.cpp](file:///mpi_test/hw_10/integrate.cpp)

### 3.1 问题与目标

使用梯形法并行计算定积分：

`∫[L,R] f(x) dx`

本实现选取函数：

- `f(x) = x^2`

### 3.2 梯形法公式

将 `[L,R]` 等分为 `N` 个小区间，`h = (R-L)/N`，则梯形法近似：

`Σ_{i=0..N-1} 0.5 * ( f(x_i) + f(x_{i+1}) ) * h`

其中 `x_i = L + i*h`。

代码中 `integrate(l, r, n)` 在局部区间 `[l,r]` 内做 `n` 个小梯形的求和。

### 3.3 区间划分（按全局小区间编号划分）

设：

- `base = N / size`
- `rem = N % size`
- `local_n = (rank < rem) ? (base+1) : base`

本地负责的小区间编号范围：

- `i_start = rank*base + min(rank, rem)`
- `i_end = i_start + local_n`

从而：

- `l = L + (R-L) * i_start / N`
- `r = L + (R-L) * i_end / N`

这样所有进程的 `[l,r]` 可无缝覆盖 `[L,R]` 且负载尽量均衡。

### 3.4 本地计算与全局规约

- 每个进程调用 `integrate(l, r, local_n)` 得到 `local_sum`
- 使用 `MPI_Reduce + MPI_SUM` 汇总得到 `global_sum`

### 3.5 计时方法

各进程计算 `TIME = END - START`，并使用 `MPI_Reduce(TIME, MPI_MAX)` 得到总耗时 `MPI_TIME`。

### 3.6 参数设置

- 积分区间：`L=0.0, R=10.0`
- 划分数：`N = 1,000,000,000`

> 注：`N=1e9` 计算量非常大，适合在资源较强的环境下测试；若用于快速验证正确性，可先用较小的 `N` 做对照。

## 4. 题目 3：并行素数统计

对应代码：[prime.cpp](file:///mpi_test/hw_10/prime.cpp)

### 4.1 问题与目标

统计 `[2, N]` 范围内素数个数，要求：

- 输出素数总数
- 输出每个进程找到的素数数量
- 对比不同进程数下运行时间

### 4.2 素数判定方法（试除法）

对每个候选数 `n`：

- 从 `i=2` 试除到 `i*i <= n`
- 若存在 `n % i == 0` 则为合数，否则为素数

该方法实现简单、便于并行拆分，但在 `N` 较大时计算较慢。

### 4.3 任务划分（循环划分）

为避免区间边界处理复杂，本实现采用循环划分：

- 每个进程检查序列：`i = rank + 2, rank + 2 + size, rank + 2 + 2*size, ...` 直到 `i <= N`

特点：

- 不会漏掉任何整数
- 各进程工作量大致均衡（**因为素数判定的开销随 `i` 增大而增加，所以不采用分块的方式不均衡**，隔size个数挑一个会更好均衡各个进程的负载）

### 4.4 汇总与输出

- 每个进程统计本地素数数量 `local_cnt`
- 通过 `MPI_Reduce + MPI_SUM` 汇总得到总素数个数 `global_cnt`（仅在 root 有效）
- 每个进程打印自己找到的素数数量；root 额外打印总数与 `MPI_TIME`

### 4.5 参数设置

- 上界：`N = 10,000,000`
