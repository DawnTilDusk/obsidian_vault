>- 而其实这道矩阵里面z字形搜索相当于把双指针的过程形象化了（l为行，r为列）
### 1. 模型的本质推演

- **指标定义**：在子区间问题中，指标是 $S[r] - S[l]$（前缀和之差）；在矩阵问题中，指标是 $Matrix[row][col]$。
- **单调性矩阵**：如果你把所有可能的子区间和 $S[r] - S[l]$ 排列成一个表格（行表示 $r$，列表示 $l$），你会发现由于 $S$ 是单调递增的，这个表格在行方向和列方向上也是自然有序的。
- **搜索本质**：
    - 在子区间问题里，双指针移动是在这个**隐形的单调矩阵**里寻找值为 $K$ 的点。
    - 在矩阵问题里，你是在一个**显性的单调矩阵**里寻找值为 $target$ 的点。

---

### 2. Z 字形搜索：双指针的形象化

“$l$ 为行，$r$ 为列”这个类比很精准。
在“和为 K 的正子区间”问题中：

- **右指针 $r$ 右移**：对应矩阵中的**向下移动**（指标变大）。
- **左指针 $l$ 右移**：对应矩阵中的**向左移动**（指标变小）。

**为什么说 Z 字形搜索是形象化？**

因为在数组双指针中，由于 $l$ 必须小于 $r$，我们其实只在矩阵的**一半（三角区域）** 里活动。而这道二维矩阵题，是给了一个完整的矩形区域让你去“对撞”。

---

### 3. 广义化：什么时候能用这种“双指针/Z字搜索”？

只要满足以下两个条件，这种 $O(M+N)$ 的策略就有效：
>[!IMPORTANT]
>1. **二维单调性**：存在两个独立的决策变量（比如行和列，或者左边界和右边界），其中一个变量增加会导致指标单调增加，另一个变量增加会导致指标单调减少。
>
>2. **决策独立性**：你可以根据当前指标与目标的差距，明确知道应该调整哪一个变量来逼近目标，且这种调整是不可逆的（即排除掉了一整行或一整列）。

### 4. 代码呼之欲出

右上角出发，每次移动筛选掉一行或者一列
如果把矩阵旋转 $45^\circ$ 看，它其实是一棵**二叉搜索树（BST）**：
- **右上角（或左下角）就是根节点**。
- 对于右上角的数：它下方的数都比它大（右子树），它左边的数都比它小（左子树）。
- **你的移动轨迹**：
    - `target > 当前值`：向下走（进入右子树）。
    - `target < 当前值`：向左走（进入左子树）。

```cpp
class Solution {
public:
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        if (matrix.empty() || matrix[0].empty()) return false;
        
        int m = matrix.size();
        int n = matrix[0].size();
        
        // 从右上角 (0, n-1) 开始搜索
        int row = 0;
        int col = n - 1;
        
        while (row < m && col >= 0) {
            int current = matrix[row][col];
            if (current == target) {
                return true; // 找到了
            } else if (current > target) {
                col--; // 当前值太大，因为这一列下方的数只会更大，所以排除掉这一列
            } else {    
                row++; // 当前值太小，因为这一行左边的数只会更小，所以排除掉这一行
            }
        }
        return false; // 越界还没找到，说明不存在
    }
};
```

## 能用分治吗？ [240. 搜索二维矩阵 II - 力扣（LeetCode）](https://leetcode.cn/problems/search-a-2d-matrix-ii/?envType=study-plan-v2&envId=top-100-liked)
 
 >[!NOTE]
>而这道题我的思路非常清奇，使用了一个二分法的二维推广
>一开始我以为可以一次**舍3求1**，后来发现漏解了很多区域
>之后发现确实只能一次舍弃1块，然后剩下三块递归讨论
### 复杂度讨论 

每次将矩阵切分为 4 块，根据与中点的比较，舍弃其中 1 块，递归搜索剩余的 **3 块**。

- **递推公式**：$T(S) = 3T(S/4) + O(1)$（其中 $S$ 是矩阵的总元素个数）。
- **计算结果**：根据主定理（Master Theorem），这个递归的时间复杂度大约是 $O(S^{\log_4 3})$，即 **$O(S^{0.79})$**。
- **直观感受**：如果矩阵是 $N \times N$，总元素 $S = N^2$，那么复杂度就是 **$O(N^{1.58})$**。这虽然比暴力遍历 $O(N^2)$ 快，但比线性$O(N + M)$ 要慢得多。

**这份代码能通过，但是实测确实慢了不少**
不过，**推广**和**化归**依然是常见的解题思路！
```cpp
class Solution {
public:
    bool find(int lx, int ly, int rx, int ry, vector<vector<int>>& matrix, int t){
        if (lx > rx || ly > ry) return false;
        if(lx < 0 || ly < 0 || rx < 0 || ry < 0
            || rx >= matrix.size() || ry >= matrix[0].size()
            || lx >= matrix.size() || ly >= matrix[0].size())
            return false;

        int mx = (lx + rx) / 2, my = (ly + ry) / 2;
        if(lx == mx && ly == my){//区域已经足够小（日字形或者田字形）
            for(int i=lx; i<=rx; i++)
                for(int j=ly; j<=ry; j++){
                    if(matrix[i][j] == t)
                        return true;
                }
            return false;
        }
        //这里是分支了三块面积，由此还引发了一些奇奇怪怪的边界判定
        if(matrix[mx][my] == t) return true;
        if(matrix[mx][my] > t)
            return find(lx, ly, mx-1, my-1, matrix, t) 
		            || find(lx, my, mx-1, ry, matrix, t)
                    || find(mx, ly, rx, my-1, matrix, t);
        if(matrix[mx][my] < t)
            return find(mx+1, my+1, rx, ry, matrix, t) 
		            || find(lx, my+1, mx, ry, matrix, t)
                    || find(mx+1, ly, rx, my, matrix, t);
        return 0;
    }
 
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        int lx = 0, ly = 0, rx = matrix.size()-1, ry = matrix[0].size()-1;
        return find(lx, ly, rx, ry, matrix, target);
    }
};
```
