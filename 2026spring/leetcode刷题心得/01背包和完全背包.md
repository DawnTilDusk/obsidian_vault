## 1. 核心定义区别

|**特性**|**01背包**|**完全背包**|
|---|---|---|
|**数量限制**|每种物品只有**一个**|每种物品有**无限个**|
|**选择决策**|选（1次）或 不选（0次）|不选、选1个、选2个...直到装不下|

---
## 2. 状态转移方程的区别

假设 $dp[j]$ 表示容量为 $j$ 的背包能装的最大价值，$w[i]$ 为重量，$v[i]$ 为价值。

- **01背包：**$$dp[j] = \max(dp[j], dp[j - w[i]] + v[i])$$
	这里的 $dp[j - w[i]]$ 必须是**上一轮（即还没放入物品 $i$ 时）**的状态。

- **完全背包：**$$dp[j] = \max(dp[j], dp[j - w[i]] + v[i])$$
    虽然形式一样，但这里的 $dp[j - w[i]]$ 可以是**这一轮已经放入过物品 $i$ 后的**状态。

---

## 3. 遍历顺序的区别（核心差异）

这是在编写代码时最容易混淆的地方。
### 01背包：必须逆序遍历容量

为了保证每个物品只被添加一次，我们在更新一维数组时，必须**从后往前**遍历。
```cpp
for (int i = 0; i < n; i++) {
    for (int j = bagWeight; j >= w[i]; j--) { // 逆序
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}
```

> **原理：** 如果正序遍历，计算 $dp[j]$ 时用到的 $dp[j-w[i]]$ 可能在同一轮里已经被更新过了（即已经放进了一个物品 $i$），这就会导致物品被重复计算。

### 完全背包：必须正序遍历容量

因为物品有无限个，我们恰恰需要利用“这一轮已经放过该物品”的状态，所以要**从前往后**遍历。

```cpp
for (int i = 0; i < n; i++) {
    for (int j = w[i]; j <= bagWeight; j++) { // 正序
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}
```

> **原理：** 正序遍历意味着在计算容量为 $j$ 的背包时，我们已经考虑过在容量更小的时候放入物品 $i$ 的情况了。

---

## 4. 总结与直观理解

- **01背包：** 像是在商店里买**限量版**，买了就没了。所以我们要从大容量往小容量推，确保参考的始终是“还没买过这件物品”的旧状态。
    
- **完全背包：** 像是在超市买**散装糖果**，拿了一个还能再拿。所以我们要从小容量往大容量推，允许在已经拿了的基础上“再拿一个”。

## 5. 关于两层循环的顺序

>[!NOTE]
>如果说**先遍历容量，再遍历物品**，这意味着对于当前容量去尝试所有物体，这个会**导致无序、重复**
>也即`1-2-3`和`3-2-1`两种组合都会因此考虑到
>所以01背包是**不可能使用**这种遍历顺序的，这里有不可调和的矛盾
>
>反之我们去理解**先遍历物品，再遍历容量**，这样会**导致有序**，
>也即如果物品1先被遍历，则所有的方案物品1都在最前面，不存在物品1莫名其妙跑到后面的情况
>如果**第二层循环从前往后**，则当前容量的决策可能会由更小的容量决定，
>就会叠加在当前的物品的决策之上，也就是用了很多个，这就是完全背包
>如果**第二层循环从后往前**，那么物品1的决策就**不会受到自己的干扰**

### 情况 1：求最大价值（最常见的完全背包）

**结论：可以交换。**

无论你是先遍历物品，还是先遍历容量，最终的结果都是一样的。

- **先物品后容量（常用）：** 逻辑是“针对当前物品，我把所有容量的背包都尝试更新一遍”。
- **先容量后物品：** 逻辑是“针对当前容量，我尝试把所有能放下的物品都试一遍”。

因为我们的目标是求**最大价值**，只要保证每个状态 $dp[j]$ 在计算时，其依赖的 $dp[j - w[i]]$ 是“当前可选范围内”的最优值即可。

---

### 情况 2：求组合/排列数（背包问题的变体）

当你做 LeetCode 上的题目（如：[518. 零钱兑换 II](https://leetcode.cn/problems/coin-change-ii/) 或 [377. 组合总和 Ⅳ](https://leetcode.cn/problems/combination-sum-iv/)）时，顺序**绝对不能**交换。
#### 1. 先遍历物品，再遍历容量 $\rightarrow$ 求的是【组合数】

这种顺序下，物品是按顺序被考虑的。比如物品是 `{1, 2}`，容量是 `3`。
逻辑会是：先处理完所有的 `1`，再处理所有的 `2`。
结果只会出现 `{1, 2}`，不会出现 `{2, 1}`。

```cpp
// 组合数代码结构
for (int i = 0; i < coins.size(); i++) {
    for (int j = coins[i]; j <= amount; j++) {
        dp[j] += dp[j - coins[i]];
    }
}
```

#### 2. 先遍历容量，再遍历物品 $\rightarrow$ 求的是【排列数】

这种顺序下，对于每一个容量 $j$，所有的物品都会被重新尝试一遍。
逻辑会是：在容量为 `3` 时，我可能先放 `1` 再剩 `2`，也可能先放 `2` 再剩 `1`。
结果会出现 `{1, 2}` 和 `{2, 1}` 两种情况。

```cpp
// 排列数代码结构
for (int j = 0; j <= amount; j++) {
    for (int i = 0; i < coins.size(); i++) {
        if (j >= coins[i]) dp[j] += dp[j - coins[i]];
    }
}
```
---
### 总结对比

|**遍历顺序**|**目的**|**结果特征**|
|---|---|---|
|**外层物品，内层容量**|求最大价值 / 求组合数|物品之间相对顺序固定，不重复计算顺序不同的集合|
|**外层容量，内层物品**|求最大价值 / 求排列数|每一层容量都重新考虑所有物品，会包含所有可能的排列|
